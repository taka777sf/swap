# 積立トラッカー（PWA）

## 更新内容（v2）
- 項目を最大5個まで登録可能に（項目名・閾値円・加算単位・単位ラベルを自由設定）
- タブで項目を切り替えて、それぞれ独立したカレンダー・繰越・週計を表示
- バックアップ機能を追加：ヘッダーの「⇩ 保存」でJSONファイルをダウンロード、「⇧ 復元」で読み込み
- 以前のバージョンでSafariに保存していたデータは、初回起動時に自動的に新項目「TRY/JPYスワップ」として引き継がれます

## GitHub Pagesへの公開手順

1. GitHub (https://github.com) にログインし、新しいリポジトリを作成
   例：`swap-calendar`（Public / Privateどちらでも可、Pagesは両方で使えます）

2. このフォルダの中身（index.html, style.css, app.js, manifest.json,
   service-worker.js, icons/ フォルダ）を丸ごとリポジトリ直下にアップロード
   - GitHubのWeb UIなら「Add file → Upload files」でドラッグ&ドロップでOK
   - 既存リポジトリを更新する場合は、同じファイル名で上書きアップロードすればOK
     （service-worker.jsのキャッシュ名をv2に上げてあるので、古いキャッシュは自動的に破棄されます）

3. リポジトリの Settings → Pages を開く
   - Source を「Deploy from a branch」に設定
   - Branch を `main` / フォルダを `/ (root)` に設定して Save

4. 数分待つと `https://<ユーザー名>.github.io/swap-calendar/` でアクセス可能になります

## iPhoneでアプリ化する

1. iPhoneのSafari（Chromeなど他ブラウザは不可）で上記URLを開く
2. 共有ボタン（□に↑）をタップ
3. 「ホーム画面に追加」を選択
4. ホーム画面にアイコンが追加され、タップするとアドレスバーなしの全画面アプリとして起動します

既にホーム画面に追加済みの場合は、一度アプリを開いて数秒待つ（バックグラウンドでService Workerが更新を取得）か、
アプリを一度削除して再度「ホーム画面に追加」し直すと確実に最新版になります。

## データとバックアップについて

- 入力データはこの端末（iPhone）のローカルストレージに保存されます
- 他の端末とは自動同期されません（同じiPhoneのSafari/ホーム画面アプリ内でのみ保持）
- **完全に消えないという保証はありません**（iOSのストレージ逼迫時の自動削除、手動でのSafariデータ消去などで失われる可能性があります）
- そのため、ヘッダーの「⇩ 保存」を定期的に（週1回など）タップしてJSONファイルをダウンロードし、
  iCloud DriveやDropbox、PCなどに保管しておくことを強く推奨します
- データが消えてしまった場合は「⇧ 復元」で直近のバックアップファイルを選択すれば元に戻せます
- 複数端末での自動同期が必要になったら、Firebase等のバックエンドに差し替えることも可能です（その場合はお声がけください）
